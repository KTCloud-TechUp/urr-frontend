# 🎫 URR (우르르)

> K-POP 찐팬을 위한 공정한 티켓팅 플랫폼

---

## 우르르가 뭔가요?

**URR(우르르)** 는 K-POP 팬이 티켓을 사고, 양도하고, 아티스트와 소통할 수 있는 **통합 팬덤 플랫폼**입니다.

기존 티켓팅 플랫폼의 가장 큰 문제는 **매크로(봇)** 와 **암표**입니다.
열심히 응원하는 찐팬이 정작 티켓을 못 구하고, 전문 리셀러나 매크로 사용자가 티켓을 싹쓸이하는 불공정한 구조를 바꾸기 위해 만들었습니다.

---

## 어떤 문제를 해결하나요?

| 기존 문제                 | URR의 해결 방법                         |
| ------------------------- | --------------------------------------- |
| 매크로·봇이 대기열을 점령 | 팬 인증 질문(VQA)으로 봇 차단           |
| 찐팬도 티켓 구하기 어려움 | 멤버십 등급에 따른 선예매 우선순위 부여 |
| 암표·외부 양도 난무       | 플랫폼 내 공식 양도 마켓만 허용         |
| 티켓팅·양도·커뮤니티 분산 | 하나의 플랫폼에서 모두 해결             |

---

## 주요 기능

### 🎟 티켓 예매

- **팬 인증 질문(VQA)**: 예매 전 아티스트 관련 질문으로 진짜 팬인지 확인 → 봇 자동 차단
- **실시간 대기열**: 공정한 순번 대기 시스템
- **멤버십 등급별 선예매**: 팬 활동이 많을수록 더 빨리 예매 가능

### 🤝 티켓 양도

- 플랫폼 내에서만 거래 → 암표 차단
- 판매자 등급·거래 횟수 공개 → 신뢰 거래
- 정가 대비 가격 표시 → 투명한 거래

### 👑 멤버십 & 등급

팬 활동을 숫자로 인정해, 더 열심히 응원할수록 더 많은 혜택을 드립니다.

| 등급        | 아이콘 | 조건             | 선예매 순서           | 양도 이용 |
| ----------- | :----: | ---------------- | --------------------- | :-------: |
| Lv.4 다이아 |   🌩️   | 팬 점수 달성     | **1순위** (가장 빠름) |    ✅     |
| Lv.3 골드   |   ⚡   | 팬 점수 달성     | **2순위** (+1시간)    |    ✅     |
| Lv.2 실버   |   ☁️   | 멤버십 가입 즉시 | **3순위**             |    ✅     |
| Lv.1 브론즈 |   🌫️   | 회원가입         | 일반 예매만           |    ❌     |

> 양도 마켓은 실버(Lv.2) 이상 멤버십 가입자만 이용할 수 있습니다.

### 💬 아티스트 소통

- 아티스트 공식 게시글 및 멤버 소식 확인
- 멤버십 가입자 전용 소통 탭

## 화면 구성 (페이지)

| 페이지        | 설명                                                    |
| ------------- | ------------------------------------------------------- |
| 홈            | 인기 아티스트, 뜨는 공연, 선예매 임박 공연 한눈에 보기  |
| 아티스트      | 팔로우 중인 아티스트 목록 및 검색                       |
| 아티스트 상세 | 아티스트 홈 / 소통 / 공연 / 양도 마켓 탭                |
| 공연 목록     | 전체 공연 일정 및 필터                                  |
| 예매 플로우   | 팬 인증 → 대기열 → 구역/좌석 선택 → 결제 → 확인 (5단계) |
| 멤버십 가입   | 아티스트 선택 → 등급 안내 → 결제 → 완료 (4단계)         |
| 마이페이지    | 내 티켓 / 멤버십 / 양도 내역 / 설정                     |

## 디자인 컨셉

- **색상**: 오렌지(`#FF5E32`) + 네이비(`#1F2792`) 투 톤
- **원칙**: 색상은 꼭 필요한 곳(등급 배지, 예매 상태, 시스템 알림)에만 사용. 나머지는 깔끔한 흑백
- **폰트**: Pretendard (본문) / JetBrains Mono (타이머·카운트다운 전용)
- **모드**: 라이트 모드 전용

## 기술 스택 (개발자용)

| 구분       | 기술                                                  |
| ---------- | ----------------------------------------------------- |
| 프론트엔드 | Next.js 15 + TypeScript + Tailwind CSS v4 + shadcn/ui |
| 상태 관리  | TanStack Query (서버 데이터) + Zustand (UI 상태)      |
| 백엔드     | Spring Boot REST API                                  |
| 인증       | JWT (카카오·네이버·이메일 OAuth)                      |
| 인프라     | AWS (Amplify/ECS + RDS + S3 + CloudFront)             |
| 아키텍처   | Feature-Sliced Design (FSD)                           |

### 로컬 실행

```bash
# 의존성 설치
npm install

# 개발 서버 실행 (http://localhost:3000)
npm run dev

# 빌드 검증
npm run build
```

> API 서버 주소: `NEXT_PUBLIC_API_BASE_URL` 환경변수로 설정 (기본값: `https://urr.guru`)

## 프로젝트 구조

```
src/
├── app/          # 페이지 라우팅 (URL 경로)
├── widgets/      # 페이지를 구성하는 큰 UI 블록
├── features/     # 로그인, 예매, 결제 등 단위 기능
├── entities/     # 아티스트, 공연, 티켓 등 데이터 모델
└── shared/       # 버튼, 인풋 등 공통 UI 및 유틸
```
