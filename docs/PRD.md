## 🎯 프로젝트 한 줄 정의

**K-POP 팬덤을 위한 통합 플랫폼**: 티켓팅 → 양도 → 커뮤니티를 하나의 생태계로 연결하여, 찐팬 중심의 공정한 거래 구조 구축

## 📋 핵심 개념

### 1) 타겟 사용자

- **찐팬(Core Fan)**: 아티스트 멤버십 가입 + 지속적인 팬 활동 (음원/MD 구매, 커뮤니티 활동)
- **배제 대상**: 매크로 사용자, 전문 리셀러, 1회성 관람객

### 2) 비즈니스 모델

- **구독 기반**: 연간 멤버십 구독료 (아티스트별)
- **수수료**: 예매/양도 시 등급별 차등 수수료

### 3) 멤버십 등급 체계 (4 Levels)

| 등급        | 조건      | 선예매 순서 | 예매 수수료 | 양도 수수료 | 양도 접근 |
| ----------- | --------- | ----------- | ----------- | ----------- | --------- |
| Lv.4 다이아 | 점수 달성 | 1순위       | +1,000원    | 5%          | ✓         |
| Lv.3 골드   | 점수 달성 | 2순위 (+1h) | +2,000원    | 5%          | ✓         |
| Lv.2 실버   | 가입 즉시 | 3순위       | +3,000원    | 10%         | ✓         |
| Lv.1 브론즈 | 회원가입  | 일반예매만  | 기본        | ×           | ×         |

### 4) 핵심 기술 메커니즘

- **VQA (Visual Question Answering)**: 매크로 방지용 보안 질문
- **Queue System**: 대기열 관리 (높은 트래픽 시 공정성 보장)
- **Official Transfer**: 플랫폼 내 양도만 허용 (외부 암표 차단)
- **Membership Score**: 팬 활동 정량화 (음원 구매, MD 구매, 커뮤니티 활동)

## 1. 프로젝트 개요

### 1.1 프로젝트 배경

K-POP 팬덤 시장은 팬더스트리(Fandustry)로 진화했으나, 티켓팅·양도·커뮤니티가 분산되고 매크로·암표가 만연하여 **찐팬이 정당한 구매 기회를 보장받지 못하는 시장 실패 상황** 발생.

**핵심 과제**

1. 매크로 및 비정상 트래픽의 구조적 제어
2. 거래 과정 및 판매 현황의 투명성 확보
3. 찐팬 중심의 참여 기회 설계

### 1.2 서비스 정의

- **서비스명**: URR | 우르르
- **서비스 유형**: K-POP 아티스트 IP 기반 팬덤 플랫폼
- **이용 방식**: 연간 멤버십 구독
- **핵심 가치**: 티켓팅 + 양도 + 소셜 커뮤니티의 End-to-End 통합 관리

### 1.3 팬더스트리(Fandustry) 정의

Fan + Industry의 합성어. 팬은 단순 소비자가 아닌 **프로슈머이자 생태계 핵심 참여자**.

> 플랫폼이 팬의 투자(시간·감정·비용)를 측정하고 멤버십 등급·티켓 우선권으로 보상해야 함.

## 2. 시장 조사

### 2.1 티켓팅 산업의 구조적 문제

**글로벌·국내 공통 문제**

- 티켓팅 플랫폼 트래픽의 상당 부분이 봇/매크로로 발생
- 기존 방어 수단(CAPTCHA, IP 차단)은 고도화된 공격에 취약
- 실제 리셀러가 정가 구매자를 선점

**국내 시장 특수성**

- K-POP 콘서트 수요 급증
- 암표/리셀은 플랫폼 외부(SNS, 중고거래)에서 비공식 진행
- 기존 플랫폼: 봇 73.5% 차단 가능하나, 구매 이후는 방치

### 2.2 사용자 리서치: 찐팬의 현실

| 항목        | 문제                         | 인사이트                        |
| ----------- | ---------------------------- | ------------------------------- |
| 선예매 참여 | 팬클럽 사전 인증 참여율 높음 | 팬 수요는 이미 공식 경로에 집중 |
| 선예매 성공 | 성공률 낮고 실패 경험 누적   | 단순 선예매로는 공정성 불가능   |
| 예매 이후   | 리셀·암표 시장 활성화        | 구매 이후 통제 구조 부재        |

**선예매의 현실**

- 전체 좌석의 30% 이상이 선예매로 우선 배정
- 선예매만으로 전석 매진되는 경우 다수 보고
- 선예매 대기번호 30,000~40,000번대 발생
- **결론**: 팬은 이미 노력 중이나, 플랫폼이 팬을 보호하지 못함

## 3. 타겟 사용자 정의

### 3.1 핵심 타겟: 아티스트 멤버십 가입 찐팬

- 연간 팬클럽 멤버십 비용 지불 중
- 티켓 실패 경험이 반복되며 페인포인트 극명
- 음원·MD 구매, 커뮤니티 활동 등 지속적 투자

### 3.2 핵심 니즈 & 페인포인트

| 니즈             | 기존 문제                      |
| ---------------- | ------------------------------ |
| 공정한 티켓 접근 | 대기열·속도 경쟁으로 반복 실패 |
| 리셀 없는 구매   | 외부 암표 시장으로 밀려남      |
| 안전한 양도      | 신뢰할 수 있는 공식 경로 부재  |
| 찐팬 커뮤니티    | 팬 인증 없는 혼탁한 커뮤니티   |

### 3.3 배제 대상

- 매크로/봇 사용자
- 전문 리셀러
- 1회성 관람객 (팬덤 활동 이력 없음)

## 4. 문제 정의 및 해결 방향

### 4.1 핵심 문제

> **멤버십 가입 찐팬조차 정당한 티켓 구매 기회를 보장받지 못하는 구조적 시장 실패**

선예매 자격을 보유한 고객이 반복적으로 구매에 실패하는 역설적 상황.

### 4.2 문제의 구조적 원인

**1) 사용자 검증 부재 → 매크로 무차별 유입**

- 선예매 대기번호 30,000~40,000번대 발생
- 봇·매크로가 대기열 다수 차지 → 정상 팬 접근 차단

**2) 불투명한 재고·거래 프로세스**

- 실시간 판매 현황 비공개
- 취소·환불·양도 프로세스 불투명
- 선예매 풀 배정 기준 비공개

**3) 분절된 팬 경험 생태계**

- 티켓팅(A사) → 양도(B사) → 커뮤니티(C사) 분산
- 사용자 행동 데이터 미연결 → 찐팬 식별 불가능
- 비정상 행위 패턴 탐지 불가능

### 4.3 해결 전략

> **티켓팅–양도–커뮤니티를 하나의 책임 구조로 통합하는 플랫폼 구축**

1. **티켓팅**: 멤버십 등급 세분화를 통한 트래픽 제어
2. **양도**: 멤버십 가입자 간 공식 양도 + 가격 가이드 + 신분 확인
3. **멤버십 등급**: 정량화된 팬 활동(음원·MD·커뮤니티)으로 티켓 우선권 차등 배분
4. **커뮤니티**: 멤버십 가입자의 인증된 양도 커뮤니티

## 5. 멤버십 분류 체계

### 5.1 멤버십 가입 프로세스

1. 회원가입 시 **브론즈(Lv.1)** 등급 부여
2. 아티스트 페이지에서 아티스트별 멤버십 가입
3. 멤버십 가입자: 티켓 예매 + 양도 모두 가능
4. 팬 활동 인증(점수) 달성 시 등업

### 5.2 등급별 혜택 (상세)

**Lv.4 다이아** (조건: 점수 달성)

- 선예매: 1순위 (가장 빠른 시간)
- 예매 수수료: +1,000원
- 양도: 수수료 5%
- 양도게시판: O

**Lv.3 골드** (조건: 점수 달성)

- 선예매: 2순위 (+1시간 늦음)
- 예매 수수료: +2,000원
- 양도: 수수료 5%
- 양도게시판: O

**Lv.2 실버** (조건: 멤버십 가입 즉시)

- 선예매: 3순위
- 예매 수수료: +3,000원
- 양도: 수수료 10%
- 양도게시판: O

**Lv.1 브론즈** (조건: 회원가입)

- 예매: 일반예매만 가능
- 예매 수수료: 기본
- 양도게시판: X
