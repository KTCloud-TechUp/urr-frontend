# 백엔드 요청 목록

프론트엔드에서 백엔드 팀에 요청할 API 변경·추가·수정 사항을 기록합니다.

> 형식: 요청 내용 / 이유 / 우선순위 (🔴 긴급 / 🟡 보통 / 🟢 낮음)

---

## 대기 중

### [🔴 긴급] `/auth/me` 응답에 실명·휴대폰 번호 필드 추가

**엔드포인트**: `GET /api/v1/users/me`

**요청 내용**: 응답 `data`에 아래 두 필드 추가

```json
{
  "userId": 1,
  "email": "user@example.com",
  "nickname": "팬닉네임",
  "name": "홍길동",
  "phoneNumber": "01012345678",
  ...
}
```

| 필드          | 타입             | 설명                                                              |
| ------------- | ---------------- | ----------------------------------------------------------------- |
| `name`        | `string \| null` | 실명 (온보딩 시 입력한 값)                                        |
| `phoneNumber` | `string \| null` | 휴대폰 번호 (SMS 인증 완료 후 저장된 값, 하이픈 없는 숫자 문자열) |

**이유**: 결제창(`PaymentView`)에서 주문자 이름·전화번호를 토스페이먼츠 SDK에 전달해야 함. 현재 사용자가 매번 수동 입력 중. `/me`에서 값을 내려주면 자동완성하여 UX 개선 가능. `null`인 경우 입력 필드를 그대로 노출해 직접 입력하도록 처리 예정.

**참고 — 현재 프론트 가입 경로별 전송 필드 차이**:

- 소셜 가입 (`POST /api/v1/auth/onboarding/social`): 실명을 `nickname` 필드에 담아 전송
- 이메일 가입 (`POST /api/v1/auth/register`): 실명을 `name` 필드에 담아 전송
- 두 경로 모두 `phone` 필드로 휴대폰 번호 전송

백엔드 DB에서 두 경로를 통일해 `name`·`phoneNumber`로 내려주면 됩니다.

### [🟡 보통] 멤버십 정책 조회 응답에 예매 구분·양도 수수료 필드 추가 - 백엔드 완료 프론트 적용해야함

**엔드포인트**: `GET /api/v1/membership/artists/{artistId}/membership-policies`

**요청 내용**: `tiers` 배열 각 항목에 아래 두 필드 추가

```json
{
  "tier": "LIGHTNING",
  "presaleOffsetMinutes": 0,
  "bookingFeeWon": 0,
  "bookingType": "PRESALE",
  "transferFeeRate": 5
}
```

| 필드              | 타입                     | 설명                                                               |
| ----------------- | ------------------------ | ------------------------------------------------------------------ |
| `bookingType`     | `"PRESALE" \| "GENERAL"` | 우선 예매 여부 (`PRESALE` = 선예매 우선권, `GENERAL` = 일반예매만) |
| `transferFeeRate` | `number \| null`         | 양도 수수료 비율(%). `null`이면 양도 불가                          |

**이유**: 멤버십 가입 화면의 "티어별 혜택 비교" 테이블에서 예매 구분·양도 수수료를 현재 프론트 하드코딩으로 표시 중. 정책 변경 시 백엔드 응답 기준으로 동적 반영하기 위해 필요.
