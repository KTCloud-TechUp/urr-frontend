## 1. 가입 및 본인 인증

### 플로우 요약

```
서비스 접속 (웹/앱)
  └─ 로그인/회원가입 선택
       ├─ 소셜 로그인 (카카오 / 네이버)
       └─ 이메일 가입 (아이디/비번 찾기 포함)
            └─ 본인 인증 단계 [1인 1계정 정책 - 매크로 방지]
                 ├─ 휴대폰 본인 확인
                 └─ 필수 약관 동의
                      └─ [완료] 가입 즉시 'Lv.1 미스트' 등급 부여
```

---

### 화면별 구성 요소

#### 로그인/가입 선택 페이지

- 서비스 로고 및 브랜드 슬로건
- 소셜 로그인 버튼 (카카오톡으로 로그인, 네이버로 로그인)
- 이메일 로그인/회원가입 버튼
- 아이디 찾기 / 비밀번호 찾기 텍스트 버튼

#### 본인 인증 페이지

- 통신사 선택 (SKT, KT, LGU+, 알뜰폰)
- 입력 필드: 성명, 생년월일(8자리), 성별/내외국인 구분, 휴대폰 번호
- 인증번호 전송 버튼
- 인증번호 입력 필드 (3분 타이머 포함)

#### 약관 동의 페이지

- '모두 동의' 전체 체크박스
- 개별 항목:
  - `[필수]` 서비스 이용약관
  - `[필수]` 개인정보 수집 및 이용
  - `[선택]` 광고성 정보 수신 동의 (앱 푸시, SMS)
- 각 항목별 상세 보기 링크 (웹뷰 또는 모달)

---

### 비즈니스 로직

| 항목                         | 설명                                                                                                                |
| ---------------------------- | ------------------------------------------------------------------------------------------------------------------- |
| **1인 1계정 원칙 (CI 기반)** | 휴대폰 인증 시 반환되는 CI(연계정보) 값을 대조하여, 기존 가입 CI가 존재하면 신규 가입을 차단하고 기존 계정으로 안내 |
| **매크로 방지 기초**         | 본인 인증을 가입 필수 단계로 배치하여 봇 계정 유입을 구조적으로 차단                                                |
| **등급 할당 로직**           | 가입 프로세스 완료 즉시 유저 테이블의 등급 상태를 `Lv.1 미스트`로 업데이트                                          |

---

### 데이터 명세 (Users Table)

| 필드명                | 타입      | 설명                                  |
| --------------------- | --------- | ------------------------------------- |
| `user_id`             | UUID      | 유저 고유 식별자                      |
| `email`               | String    | 로그인 및 연락용 이메일               |
| `auth_provider`       | Enum      | `KAKAO`, `NAVER`, `EMAIL`             |
| `ci_hash`             | String    | 본인 인증 연계정보 (1인 1계정 체크용) |
| `tier_id`             | Integer   | 현재 등급 (Default: 1)                |
| `is_marketing_agreed` | Boolean   | 마케팅 수신 동의 여부                 |
| `created_at`          | Timestamp | 가입 일시                             |

---

### 엣지 케이스

| 상황                   | 처리 방식                                                                                                             |
| ---------------------- | --------------------------------------------------------------------------------------------------------------------- |
| **인증 중복 발생**     | 이미 가입된 CI 보유 시 → "이미 가입된 계정이 존재합니다. (가입 계정: 카카오)" 팝업 노출 후 로그인 화면으로 리다이렉트 |
| **인증번호 만료**      | 3분 초과 시 → "인증 시간이 만료되었습니다. 다시 시도해 주세요." 에러 메시지 + 재전송 버튼 활성화                      |
| **미성년자 가입 제한** | 만 14세 미만 → 법정대리인 동의 프로세스로 분기 또는 가입 제한 안내                                                    |
| **네트워크 오류**      | 약관 동의 후 DB 처리 누락 시 → 재접속 시 '가입 중인 정보가 있습니다' 감지 후 약관 동의 단계부터 재시작 유도           |

## 2. 아티스트 멤버십 활성

### 플로우 요약

```
메인 홈 - 아티스트 탐색 (추천/검색)
  └─ 아티스트 홈/프로필 접속
       └─ [멤버십 가입하기 버튼]
            ├─ 아티스트별 연간 구독료 결제 (약 30,000원)
            └─ [완료] 'Lv.2 클라우드' 등급 획득 (티켓 예매/양도 권한 오픈)
                 └─ [등급 업그레이드 경로] Phase 2 연계
                      ├─ 팬 활동 인증 (음원/음반 구매 내역 등)
                      ├─ 포인트(Activity Score) 자동 정산
                      └─ [승격] Lv.3 썬더 또는 Lv.4 라이트닝 등급 달성
```

---

### 화면별 구성 요소

#### 아티스트 홈/프로필

- 현재 나의 등급 뱃지 (미스트 ~ 라이트닝)
- 다음 등급까지 필요한 점수 인디케이터
- 멤버십 가입 CTA (등급별 혜택 요약: 선예매 시간 차등, 수수료 할인)

#### 멤버십 상세/가입 페이지

- 등급별 혜택 비교표:
  - 선예매 1/2/3 순번
  - 예매 수수료 (1,000원 ~ 3,000원 할인)
  - 양도 수수료 (5% ~ 10%)
- 연간 구독 결제 수단 및 갱신 주기 안내

#### 팬 신뢰 점수(Fan Trust Score) 대시보드

- 6개 항목별 점수 분포 그래프

| 항목        | 가중치 |
| ----------- | ------ |
| 티켓 활동   | 30%    |
| 음원 소비   | 20%    |
| MD 구매     | 15%    |
| 플랫폼 활동 | 15%    |
| 신뢰 행동   | 10%    |
| 활동 기간   | 10%    |

- Gate 조건 충족 현황 체크리스트 (예매 성공 횟수, 실관람 인증률 등)

#### 활동 인증 센터

- 멜론 스트리밍/음원 구매 내역 연동 및 캡처 업로드 (음원차트 연동은 MVP 제외)
- 실물 앨범/MD 시리얼 번호 입력 및 구매 이력 확인

---

### 비즈니스 로직

| 항목                             | 설명                                                                                                                        |
| -------------------------------- | --------------------------------------------------------------------------------------------------------------------------- |
| **실시간 등급 산정 (Gate 정책)** | 점수 85점 이상이더라도 Gate 조건(예매 2회 이상, 노쇼 방지 등) 미충족 시 다이아 등급 부여 제한 → 골드로 하향 적용            |
| **티켓팅 차등 혜택**             | 유저 등급 파라미터를 티켓팅 서버로 전달하여 선예매 오픈 시간(`T`, `T+1h`, `T+24h` 등)을 제어                                |
| **수수료 차등 적용**             | 결제 단계에서 유저 등급별 수수료 변수(`fee_type`)를 다르게 호출                                                             |
| **신뢰 점수 계산 공식**          | `(티켓×0.3) + (음원×0.2) + (MD×0.15) + (플랫폼×0.15) + (신뢰×0.1) + (기간×0.1)` → 100점 만점 환산 (최근 12개월 데이터 기준) |
| **이상행동 필터링**              | 신뢰 행동 점수 1점 미만(매크로/다계정 의심) 시 → 점수와 무관하게 즉시 브론즈로 강등                                         |

---

### 데이터 명세

#### User_Artist_Status (유저-아티스트 관계)

| 필드명                  | 타입    | 설명                                         |
| ----------------------- | ------- | -------------------------------------------- |
| `current_tier`          | Integer | 미스트(1), 클라우드(2), 썬더(3), 라이트닝(4) |
| `trust_score`           | Float   | 0.00 ~ 100.00                                |
| `membership_start_date` | Date    | 가입일 (소급 계산 기준점)                    |

#### Fan_Trust_Score_Detail (항목별 점수)

| 필드명                    | 설명                       | 범위                   |
| ------------------------- | -------------------------- | ---------------------- |
| `ticket_activity_score`   | 예매 성공 및 실입장 기반   | Max 30                 |
| `music_consumption_score` | 스트리밍 및 구매 기반      | Max 20                 |
| `trust_behavior_score`    | 매크로/IP 리스크 탐지 기반 | Min 1 (1 미만 시 강등) |

#### Gate_Condition_Status

| 필드명                   | 타입    | 설명                                         |
| ------------------------ | ------- | -------------------------------------------- |
| `is_diamond_gate_passed` | Boolean | 예매 2회 & 실입장 80% 이상 등 Gate 통과 여부 |

---

### 엣지 케이스

| 상황                                 | 처리 방식                                                                                                               |
| ------------------------------------ | ----------------------------------------------------------------------------------------------------------------------- |
| **미스트 → 멤버십 가입 (점수 소급)** | 일반 회원 시절 최근 12개월 활동 이력을 가입 즉시 소급 계산 → 클라우드 아닌 썬더/라이트닝으로 시작 가능                  |
| **점수 높으나 Gate 미충족**          | 90점이더라도 실관람 인증률 낮으면 라이트닝 승급 거절 → 썬더 유지                                                        |
| **휴면 회원 복귀**                   | 멤버십 만료 후 재가입 시, 공백 기간 제외한 최근 12개월 가용 데이터로 등급 재산출                                        |
| **부정 행위 적발**                   | 티켓 되팔이 패턴 감지 → 신뢰 점수 1점 미만 하락 시 남은 멤버십 기간과 무관하게 즉시 모든 선예매 권한 박탈 + 미스트 강등 |

## 3. 등급별 차등 티켓팅

### 플로우 요약

```
공연 상세 페이지 - 예매 정보 확인
  └─ [예매하기] 버튼 클릭 (서버 시간 기준 대기열 진입)
       ├─ [권한 필터링] 유저 등급에 따른 진입 시간 통제
       │    ├─ T+0h      : Lv.4 라이트닝 선예매
       │    ├─ T+1h      : Lv.3 썬더 선예매
       │    ├─ T+2일     : Lv.2 클라우드 선예매
       │    └─ 잔여석 오픈: Lv.1 미스트 일반 예매
       ├─ [좌석 선택] 실시간 판매 현황 반영
       ├─ [결제 진행] 등급별 차등 예매 수수료 적용
       └─ [완료] 마이티켓 > 예매내역으로 티켓 발권
```

### 등급별 예매 수수료 혜택

| 등급          | 수수료 혜택           |
| ------------- | --------------------- |
| Lv.4 라이트닝 | 기본 수수료 - 3,000원 |
| Lv.3 썬더     | 기본 수수료 - 2,000원 |
| Lv.2 클라우드 | 기본 수수료 - 1,000원 |
| Lv.1 미스트   | 기본 수수료 적용      |

---

### 화면별 구성 요소

#### 공연 상세 페이지

- 공연명, 일시, 장소, 좌석도 미리보기
- 등급별 예매 오픈 타임라인 안내

#### 대기열 (Waiting Room)

- "내 앞 대기 인원: N명", "예상 대기 시간: MM분"
- 등급별 예매 접근 권한 상태 표시 (예: "썬더 등급 예매 대기 중입니다")

#### 좌석 선택 페이지

- 구역별 잔여석 실시간 카운트
- 본무대/사이드 등 구역별 필터링 기능 (KSPO DOME 등 대형 공연장 대응)

#### 결제 화면

- 티켓 가격
- 등급별 차등 예매 수수료 (-1,000 ~ -3,000원) 합계 내역

---

### 엣지 케이스

| 상황                           | 처리 방식                                                              |
| ------------------------------ | ---------------------------------------------------------------------- |
| **상위 등급 예매 시점에 매진** | 선택 가능한 좌석 없음 안내 + 취소석 대기 유도                          |
| **무통장 입금 미입금**         | 입금 기한(다음날 23:59) 내 미입금 시 자동 취소 + 해당 좌석 즉시 재유통 |

---

### 취소표(취켓팅) 대응 정책

| 등급         | 상태      | UI/UX 대응                                                                        |
| ------------ | --------- | --------------------------------------------------------------------------------- |
| **라이트닝** | 예매 가능 | 오픈 이후 상시 접속 가능. 실시간 취소표 발생 시 즉시 선택 및 결제 가능            |
| **썬더**     | 진입 차단 | 본인 오픈 시간 전까지 **[대기/카운트다운]** 화면 노출 (404 아닌 '권한 대기' 상태) |
| **클라우드** | 진입 차단 | 오픈 전까지 **[멤버십 혜택 안내/오픈 예정]** 화면 노출                            |

> **핵심 원칙:** 권한이 없는 유저가 접근 시 404 대신 '본인의 오픈 시간 안내 + 멤버십 가입 유도' 페이지를 노출하여 찐팬의 무력감을 최소화한다.

## 4. 안전 양도 및 소유권 이전

### 플로우 요약

```
[판매자]
마이페이지 > 예매내역 > 양도 등록
  ├─ 정가 가이드 준수 (비정상 리셀 차단)
  └─ 양도 게시판 리스팅 (아티스트 홈 메뉴)

[구매자]
아티스트 홈 > 양도 게시판 접속
  ├─ [권한 체크] Lv.2 클라우드 이상만 접근 가능 (미스트 차단)
  ├─ [양도 신청] 판매자/구매자 안심 인증 절차
  ├─ [결제/정산] 플랫폼 안전 결제 시스템 경유
  │    └─ 차등 양도 수수료 적용 (5% ~ 10%)
  └─ [소유권 이전] QR 티켓 재발급 및 기존 티켓 폐기
```

### 등급별 양도 수수료

| 등급          | 양도 수수료 |
| ------------- | ----------- |
| Lv.2 클라우드 | 10%         |
| Lv.3 썬더     | 5%          |
| Lv.4 라이트닝 | 5%          |

### 화면별 구성 요소

#### 판매자: 양도 등록 화면

- 보유 티켓 정보 (공연명, 좌석번호, 원가)
- 판매 가격 입력 필드: 티켓 원가로 고정(수정 불가) 또는 정가 이하만 입력 가능하도록 제어
- 양도 수수료 예상 확인: 판매 완료 시 정산받을 최종 금액 자동 계산

#### 구매자: 아티스트 홈 양도 게시판

- 양도 중인 티켓 리스트 (구역, 열 정보 노출)
- 필터/정렬: 구역별, 등록순

#### 안심 인증 및 결제 팝업

- 판매자/구매자 본인 인증 완료 마크 (CI 기반)
- 안전 결제 인터페이스: 신용카드, 간편결제 등
- 최종 결제 금액: 티켓가 + 구매자 양도 수수료

#### 소유권 이전 완료 화면

- 새로운 QR 코드가 생성된 모바일 티켓
- "양도가 완료되어 기존 판매자의 티켓은 즉시 무효화되었습니다" 안내 문구

### 비즈니스 로직

| 항목                                   | 설명                                                                                              |
| -------------------------------------- | ------------------------------------------------------------------------------------------------- |
| **폐쇄형 거래 환경 (Permission Gate)** | 미스트(Lv.1)는 게시판 조회 불가. 최소 클라우드(Lv.2) 이상만 구매 참여 가능 → 전문 리셀러 1차 차단 |
| **가격 상한제 (Price Ceiling)**        | 시스템적으로 '티켓 원가 + 플랫폼 수수료' 초과 거래 불허 → 프리미엄 거래 구조적 불가               |
| **차등 수수료 정책**                   | 클라우드 10% / 썬더·라이트닝 5%                                                                   |
| **Atomic Ownership Transfer**          | 결제 완료와 동시에 DB에서 `ticket_owner_id` 즉시 변경 + 기존 QR 폐기 + 구매자용 신규 QR 발급      |

## 5. 마이페이지 통합 관리

### 플로우 요약

```
마이페이지 (서비스 전반 모니터링)
  ├─ [마이멤버십] 가입 아티스트별 등급, 점수 상세, 활동 내역 확인
  ├─ [마이티켓] 예매/양도 내역, 거래/취소 내역 관리
  ├─ [정보 관리] 본인 인증 계정 관리 및 내 정보 수정
  └─ [고객 지원] 공지사항, FAQ, 알림 설정
```

---

### 화면별 구성 요소

#### 마이페이지 메인

- 사용자 기본 프로필 (닉네임, 대표 아티스트 등급)
- 주요 메뉴 퀵링크: 가입 멤버십(N개), 보유 티켓(N개)

#### 마이멤버십 상세 (MVP 제외)

- 가입 아티스트별 등급 카드 슬라이더
- Fan Trust Score 그래프 (6개 지표 분포도)
- Gate 조건 인디케이터: 상위 등급 승격까지 남은 핵심 조건 체크리스트

#### 마이티켓 관리

- 활성 티켓 카드: 공연 일시, 좌석 정보, 입장용 QR 코드 버튼
- 거래 히스토리: 예매 성공, 양도 완료, 취소/환불 내역 리스트

#### 정보 관리 및 설정

- 소셜 로그인 연동 정보 (카카오/네이버) 및 본인 인증 완료 상태 표시
- 푸시 알림 토글:
  - 선예매 오픈 알림
  - 양도 신청 알림
  - 아티스트 공지 알림
